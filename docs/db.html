MySqlTable ORM

A tiny, chainable PHP ORM powering expressive queries with a familiar, Laravel‑inspired API. Designed for speed and clarity while your full ORM is under development.

Production‑ready reads Writes are basic Eager loading (link)

Installation

Drop the MySqlTable class into your project and provide a global $db with a getConnection() method that returns a mysqli connection. Models should extend this class.

<?php
// db.php
class DB { public function getConnection() { /* return mysqli */ } }
$db = new DB();

// Base model
abstract class BaseModel extends MySqlTable {
    public function __construct() { parent::__construct(static::$table); }
    public function find($id) { return $this->where(['id' => $id])->first(); }
}

class AccountModel extends BaseModel { protected static $table = 'accounts'; }
?>

Quickstart

$accounts = model('AccountModel')
    ->select(['id','name'])
    ->where(['status' => 'active'])
    ->sort_by('created_at','DESC')
    ->limit(10)
    ->get();

$first = model('AccountModel')->where(['email' => 'a@b.com'])->first();

API Reference

Constructor

__construct(string $table) — sets the target table and grabs a mysqli connection from global $db.

new MySqlTable('users');

select()

Choose columns. Accepts string or array. Defaults to *.

->select(['id','name'])

where()

Add ANDed equality conditions from an associative array.

->where(['status' => 'active', 'account_id' => 7])

group_by()

Group results by a column.

->group_by('role')

sort_by()

Order by column and direction (ASC|DESC).

->sort_by('created_at','DESC')

limit()

Limit the number of rows returned.

->limit(25)

get()

Run the built SELECT and return an array of row objects. Resets the builder.

->get()

first()

Convenience for a single row (adds LIMIT 1).

->first()

insert()

Insert a row and return the inserted object (by re‑querying with the new ID).

$user = model('UserModel')->insert([
  'name' => 'Ali',
  'email' => 'ali@example.com',
]);

update()

Update based on current where() conditions. Returns boolean.

model('UserModel')
  ->where(['id' => 5])
  ->update(['name' => 'New Name']);

delete()

Delete rows matching the current where(). Returns boolean.

model('UserModel')->where(['status' => 'inactive'])->delete();

count()

Return integer count of rows for current criteria.

$pending = model('TxnModel')->where(['status' => 'pending'])->count();

all()

Fetch all rows from the table (resets internal state first).

$users = model('UserModel')->all();

rawQuery()

Execute a raw SQL statement and get an array of associative arrays.

$rows = model('UserModel')->rawQuery('SELECT id, email FROM users WHERE email LIKE "%@example.com"');

Eager Loading with link()

Preload nested relationships using a dotted path. For each relation segment, the ORM expects a foreign key named <relation>_id on the current object and resolves it using a model named Ucfirst(relation)Model. Relations are attached as nested properties.

$txns = model('TransactionModel')
  ->link('account.user')
  ->where(['status' => 'pending'])
  ->sort_by('created_at','DESC')
  ->get();

// Accessing
$txns[0]->account; // Account object
$txns[0]->account->user; // User object

How it works

  1. get() executes the main query and builds row objects.
  2. For each path in link(), it recursively resolves segment_id via model('{Segment}Model')->find().
  3. Resolved models are attached to the parent object under the segment name.

Controller Example

<?php
class DashboardController
{
  public function adminDashboard($request, $response)
  {
    $accounts = model('AccountModel')->all();

    foreach ($accounts as $account) {
      $account->cards = model('CardModel')
        ->where(['account_id' => $account->id])
        ->get();

      $account->apps = model('AppModel')
        ->select(['id','name','status'])
        ->where(['account_id' => $account->id])
        ->get();
    }

    $pending_txns = model('TransactionModel')
      ->link('account.user')
      ->where(['status' => 'pending'])
      ->sort_by('created_at','DESC')
      ->get();

    return $response->view('admin.dashboard', [
      'user' => $request->user,
      'accounts' => $accounts,
      'pending_txns' => $pending_txns
    ]);
  }
}
?>

Best Practices & Notes

State Management

Each terminal call (get(), first(), update(), delete(), count(), all()) resets the builder via reset(). Chain fresh conditions after each call.

SQL Safety (Important)

Current implementation concatenates values into SQL. Prefer prepared statements with bound parameters to avoid injection and quoting issues. Plan to migrate where(), insert(), and update() to bindings.

Naming Conventions

Relations expect segment_id fields and corresponding {Segment}Model classes with a find($id) method.

Pagination

Use limit() + sort_by() for simple pagination. A dedicated paginate() helper is on the roadmap.
